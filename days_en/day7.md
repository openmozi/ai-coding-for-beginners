## 2. Good vs Bad Review

**Bad Review**: "This won't work"

**Good Review**: "I see your new method matches the existing style... Having that many parameters hurts readability..."

## 4. What AI Changed

- **Efficiency**: Scan large codebases quickly
- **Consistency**: Apply same standards
- **Knowledge Sharing**: Spread best practices
- **Reduced Cognitive Load**: Automate routine checks

---

## 5. AI Code Review Limitations

- More configuration/setup
- False positives
- Can't handle complex logic (needs human)
- Security changes need extra caution

> **Important**: You own the code that is merged. Can't blame AI.
