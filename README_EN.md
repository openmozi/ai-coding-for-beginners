## Course Introduction

In the era of Large Language Models, software development is undergoing a revolutionary change. The traditional software development lifecycle is being reshaped by AI automation. This course will help you master modern AI programming tools, understand best practices for AI-assisted development, and prepare for a future career in software engineering.

### Course Information

- **Course Code**: CS146S
- **School**: Stanford University
- **Semester**: Fall 2025
- **Credits**: 3 units
- **Website**: https://themodernsoftware.dev

### Prerequisites

- CS111 equivalent programming experience
- CS221/229 recommended (but not required)

### Course Format

- Weekly lectures
- Hands-on programming sessions
- Industry guest speakers
- Final project presentations

### Course Objectives

- Master modern development tools
- Understand AI-assisted programming
- Learn automated testing and deployment
- Explore emerging software engineering trends

### Grading

| Component | Weight |
|-----------|--------|
| Final Project | 80% |
| Weekly Assignments | 15% |
| Class Participation | 5% |

## Featured Content

### Anthropic Internal Practices

This tutorial integrates **How Anthropic Teams Use Claude Code** reading material, featuring real-world experience from Anthropic's internal teams:

| Team | Claude Code Use Cases |
|------|----------------------|
| **Data Infrastructure** | Kubernetes debugging, data workflow automation, codebase navigation for new hires |
| **Product Development** | Product feature development, bug fixes, refactoring |
| **Security Engineering** | Security code review, vulnerability fixes, threat modeling |
| **Data Science** | Data analysis, visualization, experiment analysis |
| **API Team** | API documentation generation, SDK development |
| **Growth Marketing** | A/B test analysis, marketing automation |
| **Product Design** | Design system documentation, prototype iteration |
| **RL Engineering** | Reinforcement learning training pipelines |
| **Legal** | Contract analysis, workflow automation |

### Key Best Practices

1. **Detailed Claude.md files** - Document workflows, tools, and expectations
2. **Use MCP servers** - Extend Claude Code capabilities
3. **Screenshot assistance** - Use screenshots to show expected interfaces
4. **Incremental development** - Implement one step at a time
5. **End-of-session documentation** - Summarize completed work, improve workflows
