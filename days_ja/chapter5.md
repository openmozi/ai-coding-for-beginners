## 2. AI開発プロダクト7大原則

### 原則1: Start with what developers know
開発者が既に知っていることから始める

- 既存のインターフェースからの移行を促進する
  - 誰もがIDEを使う → Cursor
  - ターミナルは理解しやすいユーザー体験 → Warp
  - 人々はチャットに慣れている → Bolt
- コードと自然言語の簡単な切り替え

### 原則2: Configuration flexibility
設定の柔軟性

- **ゼロセットアップ**: 一般ユーザーに即座に価値を提示
- **シームレスなモデル切り替え**: 異なるモデル間で簡単に切り替え
- **上級ユーザー向けの高度なカスタマイズ性**:
  - Prompts
  - Project rules
  - MCP

### 原則3: Focus on developer ergonomics
開発者エルゴノミクスに注目する

- **すべてのキーストロークを節約** - キーストロークを1つでも省けるなら、実行する
  - キーボードショートカット
- **ゼロオンボーディング摩擦**
  - "5 minutes to WOW"

### 原則4: Chat as a first-class citizen
チャットをファーストクラスシチズンとする

- コードは本質的に人間の意図の人為的な表現(contrived representation)である
- ツールが進化するにつれて、より多くのことが自然言語で直接表現されるようになる
- 進化の方向性: 構文ではなく、開発者の意図をシンプルに伝える

### 原則5: MCP Integration
MCP統合

- MCPはLLMと現実世界を相互作用させるlingua franca(共通言語)となった
- 拡張可能なツールエコシステムで、LLMがあらゆるリソースにアクセスし、あらゆることを実行できる

### 原則6: Rapid feedback loops
高速フィードバックループ

- 変更を迅速に反復し、更新を確認できる
- パネルとダッシュボードがプロンプトに応じてほぼリアルタイムで変化
- これがプロジェクト全体がビルド時間の削減に専念している理由
- **説明可能性がファーストクラスシチズン**

### 原則7: Agent Workflows
エージェントワークフロー

- 実質的なタスクの完全自律
- "agent-take-the-wheel"アプローチの採用が増加
- 異なるレベルの人間とエージェントの協働:
  - エージェントが明確化のための質問をする
  - YOLO(完全自律)
